# Security Policy

## Supported versions

Security fixes are applied to the latest `master` branch.

## Reporting a vulnerability

Please report suspected vulnerabilities privately by email:

- **f.fer@institut-myologie.org**

Include:

- a clear description of the issue,
- impact and affected components,
- reproduction steps or proof of concept,
- suggested mitigation if available.

Do not open public issues for unpatched vulnerabilities.

## Response process

1. Acknowledge receipt.
2. Reproduce and assess severity.
3. Prepare and validate a fix.
4. Publish a coordinated patch and advisory notes.
